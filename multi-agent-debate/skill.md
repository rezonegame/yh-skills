# Multi-Agent Debate / 多视角角色讨论

通过精心设计的提示词，让同一AI模型扮演不同身份的角色进行多轮对话讨论复杂话题。

---

## 🤖 执行流程

### 第1步：解析讨论主题

从用户输入中提取讨论主题。如果主题不够清晰，先通过 AskUserQuestion 工具明确讨论范围和焦点。

### 第2步：选择参与者角色

从以下15个角色中选择5-8个参与讨论：

**专家角色（8位）**：
- 🎮 游戏设计师 - 用户体验与互动设计
- 🎯 产品经理 - 商业可行性与市场需求
- 💼 技术架构师 - 技术实现与可行性
- 📚 教育学家 - 学习科学与教育应用
- 🔍 批评家 - 风险识别与批判性分析
- 📊 数据科学家 - 数据驱动与量化分析
- ⚖️ 法务专家 - 法律合规与监管风险
- 🌱 环保主义者 - 可持续发展与社会影响

**普通人角色（7位）**：
- 👨‍🎓 普通学生 - 学习者视角与实际体验
- 👩‍🏫 普通老师 - 教学一线与实践挑战
- 🫣 低调的人 - 隐私保护与个人权益
- 🤔 转专业生 - 补课需求与支持资源
- 💼 职场新人 - 职场应用与实际价值
- 👴 老年人 - 适老化与可及性
- 🧑‍💼 创业者 - 创业机会与商业价值

**选择策略**：
- 确保至少包含3位专家角色和2位普通人角色
- 选择与主题最相关的角色
- 确保角色视角的多样性

选择完成后，向用户展示选定的角色列表。

### 第3步：询问是否需要调研

使用 `AskUserQuestion` 工具询问用户：

```
🤔 这个话题是否需要先进行调研？

调研可以让每个角色从自己的视角收集背景信息，
提升讨论的深度和多样性。

选择：
- **需要调研** - 深入背景分析（推荐复杂话题）
- **直接讨论** - 基于现有知识开始
```

如果用户选择"需要调研"，执行第4步（调研阶段）。
如果用户选择"直接讨论"，跳到第5步（第1轮讨论）。

### 第4步：差异化调研（如用户选择）

```
🔍 第0阶段：差异化调研

我会让每个选定的角色从自己的独特视角调研这个话题。
调研完成后，将结果整合为背景材料供讨论使用。
```

**调研执行方法**：
- 为每个角色生成2-3个特定的调研问题
- 使用 WebSearch 或 Task 工具收集相关的背景信息、数据、案例
- 整合为结构化的调研背景材料

**调研完成标志**：向用户展示调研背景摘要，确认进入正式讨论。

### 第5步：执行三轮讨论

直接在对话中进行三轮讨论，每轮都模拟不同角色的发言。

#### 🎯 第1轮：初始观点

为每个角色生成初始观点发言。

**格式示例**：
```markdown
## 第一轮：初始观点

### 🎮 游戏设计师

从用户体验和互动设计的角度...

### 🎯 产品经理

从商业可行性和市场需求的角度...
```

**关键要点**：
- 每个角色从自己的专业/个人角度分析话题
- 保持角色视角的独特性
- 使用角色特定的语言风格

#### ⚔️ 第2轮：互相辩论

让角色之间互相回应和辩论。

**格式示例**：
```markdown
## 第二轮：互相辩论

### 🎮 游戏设计师 → 回应 🎯 产品经理

你提到的市场需求确实重要，但我认为...

### 📚 教育学家 → 回应 💼 技术架构师

技术实现是基础，但我们也需要考虑...
```

**辩论策略**：
- 选择有趣的互动配对
- 让角色承认对方观点并调整立场
- 创造真实的对话冲突

#### 💡 第3轮：反馈与总结

角色对之前的辩论进行反馈，并形成综合判断。

**格式示例**：
```markdown
## 第三轮：反馈与总结

### 🔍 批评家

经过刚才的讨论，我对这个话题有了新的认识...

### 💡 综合总结

基于以上三轮讨论，我们可以得出以下结论：
1. 核心共识...
2. 主要分歧...
3. 关键洞察...
4. 可执行建议...
```

### 第6步：保存结果

将讨论结果整理为Markdown格式并保存：

**文件1：过程记录** - `讨论_过程记录_YYYYMMDD-HHMMSS.md`
**文件2：总结报告** - `讨论_总结报告_YYYYMMDD-HHMMSS.md`

使用 Write 工具将内容保存到当前目录。

---

## 💡 执行建议

**讨论质量控制**：
- 每轮讨论选择3-5个最相关的角色，避免展示所有角色
- 让角色之间有真实的互动，包括承认对方观点、调整立场等
- 保持讨论的自然流畅，避免机械化
- 最终总结要有价值，提炼出可执行的洞察和建议

**调研阶段优化**：
- 调研阶段可以简化，主要提供关键背景信息
- 不需要过度详细，聚焦于支持讨论的核心信息
- 每个角色的调研方向应该有显著差异

**角色模拟技巧**：
- 参考角色提示词模板（见下方参考文件）
- 专家角色：专业性、独特视角、专业但易懂的语言
- 普通人角色：口语化、真实情绪、说实话
